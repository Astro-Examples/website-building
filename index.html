Session 3.a | Building a Website

Today we're going to be building a website in several stages. To start with this folder, run:

git clone https://github.com/Astro-Examples/website-building.git in a directory of choice.

Goals

  • Cover the basic files and structuring of a website that uses HTML, CSS, and (sometimes) a bit of JavaScript.
  • Learn the basic syntax for html, such as Italics, bold, headings, links, and images.
  • Learn the basics of syling with CSS.
  • Learn how to import and use simple elements from the Bootstrap Library
  • Finally, how to take an html/css theme, and replace/modify things until it is a site you are happy with.

Following Along

You can edit the html/css files in whatever text editor you prefer. To visualize the webpage you're creating, you can simply double click to open in your default web browser. When you save changes to the file, refreshing the page (e.g., cmd+r or ctrl+r) will show your changes.

Alternatively, to use VSCode as I am here, install the "Live Preview (Microsoft)" extension to VSCode. If you do, you'll see changes to the file happen in real time.

Part I: Structure

Everything in HTML is contained within some type of open-close tag, much like in latex (e.g., \begin{equation} \end{equation}). These tags provide the information about how to style or format the content contained within. HTML tags are contained within< > symbols, with the closing tag being the forward slash plus the name (e.g., < i>this would be italics< /i>).

From a structural standpoint, the primary html "container" tag is the div tag(<div>). Divs can be used to contain any block of content for which a particular set of styles and formattings should be used. These can be set on the fly... for example, you can add an "align" parameter to a div, like so: <div align='center'></div>

This is a center aligned div!

As soon as I start writing after the close of the div, I revert to whatever the default style was outside of the div.

We can use divs to immediately style this 1980s-ass looking website to look a little nicer. My first aesthetic concern is that our text is filling all the way to the edges of the page.

To fix this, we will create a new Class. By creating a class (in our CSS file) we can tell a div to use that style, and we can specify some padding for our site there.

On modern websites, divs are used to separate and set stylings for almost everything. Most likely, you won't be writing a site from scratch, you'll be using a theme, or elements, to fill out your site. Being familiar with divs and how they work will allow you to identify them in these themes and modify them as necessary.

Part II: Styling

Like other markup languages, styles are applied with tags. Where markdown uses *asterisk* for italics, html uses < i>bracket tags< /i>). The standard ones in html are:

  • i: italics
  • b: bold
  • u: underline
  • strong: strong
  • a href="": links
  • img src="": images
  • ul: unordered list
  • ol: ordered list
  • li: list element
  • h1/h2/h3: headers
  • More: here's a list.
Anything more complicated with these is handled via CSS with exact specifications.

Part III: CSS

CSS (cascading style sheets) are a way to overwrite the default style of html, as well as define new classes with specific styles and then use them in our sites.

Again, when building out of other elements (like bootstrap), or in a theme (most themes use bootstrap), we are not interested in learning css from scratch (it's extensive), but more in finding where in a css file a parameter we want to change is, and learn how to change it.

Part IV: Bootstrap

Here we'll modify our site by using bootstrap elements. It is possible to build a decent looking site from scratch using just the styles, containers, and components available in bootstrap. Beyond that, almost every freely available template/theme uses these elements extensively, so being loosely familiar with them will help you modify template files a lot more efficiently.
